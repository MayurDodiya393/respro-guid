# Respro Dashboard — Master User Guide

## Introduction

The **Respro Dashboard** is a modern web-based business management system designed to make daily operations easier, faster, and more organized.

we are currently manage work using Google Sheets, WhatsApp messages, manual follow-ups, and phone calls. While these methods work, they often create challenges such as:
- Missing updates
- Manual reminders
- Duplicate work
- Difficult tracking
- No proper accountability
- Hard-to-manage task history

Respro Dashboard provides a better and more organized way to manage work through one connected system.

> Respro Dashboard is not intended to replace Google Sheets completely.  
> Instead, it provides a smarter and more efficient way to manage operational work, task tracking, follow-ups, workflows, and team coordination.

# Main Modules of Respro Dashboard

The Respro Dashboard is divided into three major systems:

1. FMS (Flow Monitoring System)
2. Checklist Engine
3. Delegation Engine

All these systems work together inside one dashboard.

---

# 1. FMS (Flow Monitoring System)

## What is FMS?

FMS is used for business processes that move step-by-step between different people or departments.

### Example:
- Purchase Request
- Manager Approval
- Accounts Verification
- Final Completion

Instead of manually calling or messaging the next person, the system automatically moves the work forward.

---

## Main Features of FMS

### Step-by-Step Workflow
Each process is divided into proper steps.

### Automatic Task Assignment
The system automatically assigns work to the correct person or role.

### Workflow Tracking
Managers can see:
- Pending work
- Completed work
- Delayed work
- Current process status

### Smart Notifications
Users receive notifications when work is assigned or delayed.

### Dynamic Forms
Different workflows can collect different information without creating new systems every time.

### SLA & Due Date Tracking
The system tracks planned dates and delays automatically.

### Audit History
Every activity is recorded for future reference.

---

# 2. Checklist Engine

## What is the Checklist Engine?

The Checklist Engine is used for repeated daily, weekly, monthly, or yearly operational tasks.

### Examples:
- Daily site inspection
- Weekly stock check
- Monthly maintenance
- Safety audits

Instead of creating the same tasks repeatedly in spreadsheets, the system generates them automatically.

---

## Main Features of Checklist Engine

### Automatic Recurring Tasks
Tasks can repeat:
- Daily
- Weekly
- Monthly
- Quarterly
- Yearly

### Smart Scheduling
The system can:
- Skip holidays
- Skip weekends
- Move tasks to working days

### Automatic Reminders
Users receive reminders before due time.

### Escalation System
If tasks are delayed, managers can be notified automatically.

### Score System
Performance scores are generated automatically based on:
- On-time completion
- Delays
- Missed work

### Historical Records
Completed work history remains stored for future review.

### Self-Healing Task Generation
If any task is missing, the system can regenerate it automatically.

---

# 3. Delegation Engine

## What is the Delegation Engine?

The Delegation Engine is used for assigning one-time follow-up tasks between people.

### Examples:
- Manager assigning corrective action
- Coordinator assigning follow-up work
- Team member revision requests
- Deadline extension management

---

## Main Features of Delegation Engine

### Task Delegation
Managers or coordinators can assign tasks directly to users.

### Revision Management
Deadlines can be revised when needed.

### Leave Adjustments
Date changes due to leave can be handled separately.

### Weekly Performance Scores
The system calculates weekly performance automatically.

### Follow-Up Monitoring
Managers can monitor:
- Pending tasks
- Overdue tasks
- Revised tasks
- Completed tasks

### Status Tracking
Task statuses include:
- Pending
- In Progress
- Completed
- Overdue
- Revised
- Cancelled
